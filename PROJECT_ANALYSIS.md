
# 대대손손 프로젝트 현황 분석 및 개선 제안서

## 1. 현재 상태 진단 (Current Status)

### 🏗 아키텍처 (Architecture)
- **Framework**: Next.js 14 (App Router)
- **Database**: `data/facilities.json` 파일 기반 단순 저장소. (Prisma가 설치되어 있으나 미사용 상태)
- **Hosting**: Vercel
- **Styling**: Mantine UI + Tailwind CSS (Utility)

### ✅ 잘 구현된 점 (Pros)
- **지도 연동**: 네이버 지도 API 연동 및 마커/클러스터링 UI가 우수함.
- **상세 UI**: 모달, 사이드 패널 등 최신 호갱노노/네이버부동산 스타일의 UX가 적용됨.
- **검색/필터**: 기본 키워드 검색 및 카테고리 필터링 동작.
- **관리자 기능**: PDF 파싱을 통한 데이터 자동 입력, 가격 정보 수정 기능 보유.

### ⚠️ 미비된 점 및 위험 요소 (Cons & Gaps)
1.  **데이터 무결성 문제**:
    - 파일 시스템(`json`)에 의존하고 있어, 동시 접속자가 많거나 배포 환경(Serverless)에서 데이터가 초기화/유실될 위험이 큼.
    - *개선 시급*: Supabase 또는 SQLite/Postgres(Prisma)로의 정식 마이그레이션 필요.
2.  **보안 취약점 (Security)**:
    - `/admin/upload` 페이지가 별도 인증 없이 공개되어 있음. 누구나 데이터를 수정/삭제 가능.
    - *개선 시급*: 관리자 로그인(Auth) 기능 도입.
3.  **SEO 및 마케팅**:
    - 검색엔진 최적화(SEO)를 위한 `sitemap.xml`, `robots.txt` 부재.
    - 오픈그래프(OG) 태그 디테일 부족 (카카오톡 공유 시 이미지/설명).
4.  **사용자 경험 (UX)**:
    - **로그인/회원가입 부재**: 리뷰를 비회원으로만 작성 가능하여 신뢰도 하락 및 어뷰징 우려.
    - **찜하기/최근본내역**: 휘발성 데이터로 사용자 재방문 유도 기능 부족.

## 2. 호갱노노 수준의 서비스를 위한 로드맵 (Roadmap)

### 1단계: 필수 기반 마련 (Current Priority)
- [ ] **SEO 최적화**: `sitemap.ts`, `robots.ts` 생성 및 메타데이터 동적 생성 강화.
- [ ] **관리자 보안**: `/admin` 경로 진입 시 비밀번호 확인 기능 추가.
- [ ] **Google Analytics**:방문자 분석 도구 연동.

### 2단계: 데이터 안정화
- [ ] **DB 마이그레이션**: JSON 데이터를 Supabase(Postgres)로 이관.
- [ ] **API 최적화**: 파일 입출력 대신 DB 쿼리로 변경하여 속도 10배 이상 향상.

### 3단계: 커뮤니티/회원 기능
- [ ] **소셜 로그인**: 카카오/네이버 로그인 연동.
- [ ] **실명 리뷰 시스템**: 로그인 사용자만 리뷰 작성.
- [ ] **마이페이지**: 찜한 시설, 내가 쓴 리뷰 관리.
